## Copy imx447 driver into kernel

Can be found in rpi kernel: https://github.com/raspberrypi/linux
More speificallty: https://raw.githubusercontent.com/raspberrypi/linux/refs/heads/rpi-6.12.y/drivers/media/i2c/imx477.c

Download to appropriate directory in linux-imx kernel:
$ cd drivers/media/i2c
$ wget https://raw.githubusercontent.com/raspberrypi/linux/refs/heads/rpi-6.12.y/drivers/media/i2c/imx477.c

Make sure it is there:
$ ls | grep imx477
Should show `imc477.c`

## Edit the driver

include the following:

#include <linux/media-bus-format.h>

and

#ifndef MEDIA_BUS_FMT_SENSOR_DATA
    #define MEDIA_BUS_FMT_SENSOR_DATA MEDIA_BUS_FMT_META_14
#endif

https://www.kernel.org/doc/html/latest/userspace-api/media/v4l/subdev-formats.html

## Edit Kconfig and Makefile

Open drivers/media/i2c/Kconfig (its a file)
Add an entry for the IMX477 driver. Like:

A guess is to use the one from the raspberrypi kernel, as the other entries are identical:
https://github.com/raspberrypi/linux/blob/rpi-6.12.y/drivers/media/i2c/Kconfig

config VIDEO_IMX477
	tristate "Sony IMX477 sensor support"
	depends on I2C && VIDEO_DEV
	select V4L2_CCI_I2C
	select VIDEO_V4L2_SUBDEV_API
	select MEDIA_CONTROLLER
	select V4L2_FWNODE
	help
	  This is a Video4Linux2 sensor driver for the Sony
	  IMX477 camera. Also supports the Sony IMX378.

	  To compile this driver as a module, choose M here: the
	  module will be called imx477.

Open drivers/media/i2c/Makefile (it is a file) and add:
obj-$(CONFIG_VIDEO_IMX477)  += imx477.o

## Configure kernel

(run make ARCH=arm64 imx_v8_defconfig for def config. Not nessesary at the start, but if you fuck something up it can be.)

In linux-imx (kernel root)
$ make ARCH=arm64 menuconfig

Activate RKISP1 driver:
In menuconfig: Device Drivers ---> Multimedia support ---> Media drivers ---> Media platform devices
Find: "Rockchip Image Signal Processing v1 Unit driver" and include it as a module <M> (press space), or as built-in <*> (if possible I guess, I don't know the spesifics)
Under "Media platform devices"" several NXP medai platform drivers are also present. (dw 1000 dewarper, i.mx8 ISP, i.mx pxp, nxp mipi csi-s csis receiver (for i.mx7 and 8 models))
Anything that is related to: Enable V4L2, Media Controller Framework, and other dependencies (often enabled by default): CONFIG_MEDIA_CONTROLLER, CONFIG_V4L2_MEM2MEM_DEV (or CONFIG_V4L2_MEM2MEM_DRIVERS), CONFIG_VIDEO_DEV, Any Rockchip or NXP i.MX8MP platform-specific modules needed for camera/CSI.

Activate newly incooporated imx447 i2c ancillary driver!:
In menuconfig: Device Drivers ---> Multimedia support ---> Media ancillary drivers ---> Camera sensor devices

SAVE THE CONFIG
(if you feel like you have fucked something up, and want to start over, run "make defconfig". This makes a default config)

Look into ./.config and check that is has all the options you hoped
I reccomend to grep, but you do you
Find stuff like:
CONFIG_VIDEO_ROCKCHIP_ISP1=y
# or =m for module, but =y is recommended for embedded
CONFIG_MEDIA_CONTROLLER=y
CONFIG_V4L2_MEM2MEM_DEV=m
CONFIG_VIDEO_DEV=y
CONFIG_VIDEO_IMX477=y

# COMPILE TIME

Hopefully you have all the deps (gcc/clang, make, bison, flex, openssl-dev, elfutils, etc.)

If you are compiling on the board itself it is straigt forward, simply run:
$ make -j"$(nproc)" Image modules dtbs
This will take forever

If you are smart and on a PC, you can crosscompiel to the arm64 architecture. In bash run:
make -j"$(nproc)" Image modules dtbs ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu-
This takes a very much shorter forever
